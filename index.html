Monica

Monica is a Go project that helps developers avoid repeating commands by defining a structured .monica.yml config file.

Installation

sudo curl -sSo /usr/bin/monica https://raw.githubusercontent.com/CapSens/monica/master/monica && sudo chmod 777 /usr/bin/monica

Example of use

Let's say we need to type almost everyday the following commands:

  • rake assets:clobber assets:precompile
  • git add -A
  • git commit -m 'A commit message that nobody will care about'
  • git push origin master

Here is what the .monica.yml file would look like:

engine: monica
reactions:
  - name: push
    desc: Pushing current branch to ${b} on Gitlab
    content:
      - command: git add -A
      - command: git commit -m '${m}'
      - command: git push origin ${b}

  - name: com-push
    desc: Precompiling assets and pushing to Gitlab
    content:
      - command: rake assets:clobber assets:precompile
      - reaction: push #Reusing reaction above with name 'push'

The config file should be placed at the root of the git repository to be detected and parsed by monica. Once done, you can call the following command :

monica push -b master -m "commit message"

or to precompile and push using commit message with staging branch:

monica com-push -b staging -m 'commit message'
